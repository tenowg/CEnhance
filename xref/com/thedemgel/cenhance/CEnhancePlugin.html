10  /**
11   * Define structure of Skill/Abilty/Effect search.
12   * 
13   * Use Annotations to get passive modifiers for Skills/Abilities/Effects
14   * 
15   * Modifiers
16   *   Effects/Skills/Abilities can add to Modifiers list.
17   *   ConcurrentMap (Modified (Class), ConcurrentMap(modifierId, modifier)
18   *   When skill is added, look for @Modifier on Modifier fields
19   *   when skill is removed, reverse process
20   * 
21   *   Events fired on addition/removal of Modifiers/Skills/Effects
22   * 
23   *   ModifierHandler
24   * 
25   *   Modifier class
26   *     modifierId = UUID or Long
27   *     One method (modify) - returns Object with proper modifier
28   * 
29   * Effects can stack: options
30   *   Effect container - map and sorted list
31   *     Sorted List is descending from time left of effect
32   *     Map effectid/Effect - effectid = UUID or Long
33   * 
34   *   Effect Object
35   *     stack = how much it can stack (default)
36   *     getpossiblestack will calculate stack amount possible at moment
37   *     execute will perform the effect
38   * 
39   * Skills Cannot be stacked
40   *   Map skillid/Skill - skillid = UUID or Long
41   * 
42   *   Skill Object
43   *     Cooldown = long milliseconds
44   *     execute will perform the skill.
45   * 
46   *   Skills can add passive Modifiers, or active Effects
47   * 
48   * Map (TypeofAbility (Enum?)/Map("stringid"/Ability-Skill))
49   *   Type is Class Annotation and stringid is class annotation
50   */
